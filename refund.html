처리 전 확인

환불과 취소는 금액과 행정 처리 영향이 커서 신청 화면보다 한 번 더 확인하는 흐름이 중요합니다.

  • 취소 유형이 수강취소인지 기타인지 먼저 구분
  • 환불마감 여부를 강좌명 아래 상태와 함께 확인
  • 학생 정보와 보호자 연락처를 같이 확인
  • 신청일과 처리일을 보고 시점 차이를 점검

금액 영향 확인

수강료와 수용비, 재료비는 환불 계산에 직접 연결되므로 한 줄씩 읽히게 하는 편이 안전합니다.

  • 괄호 안 금액은 부분 환불 또는 반영 금액으로 확인
  • 재료비, 교재비는 별도 기준이 있을 수 있어 함께 검토
  • 환불 데이터 삭제만으로 수강료는 바뀌지 않는 점 주의
  • 필요하면 신청자관리에서 원 수강료를 다시 확인

상태별 처리 흐름

접수, 강사확인, 처리완료처럼 상태가 섞이기 쉬워서 지금 어떤 단계인지 먼저 보여주는 편이 좋습니다.

  • 처리완료는 행정 제출 전 최종 결과 검토용
  • 강사확인 단계는 추가 확인 요청이 필요한 구간
  • 취소 가능 여부는 접수 상태와 함께 판단
  • 삭제 전에는 실제 금액 수정 필요 여부를 같이 확인
